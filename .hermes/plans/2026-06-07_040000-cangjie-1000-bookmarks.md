## 完整要求

### 命名规则
- `.cj` 文件名 = 有意义的英文名（如 `azureOpenaiChatgpt.cj`）
  - **不能**用 `entry-1`、`entry-2` 这种无意义命名
- slug（路由名）和 `.cj` 文件名一致
  - 如 `azureOpenaiChatgpt.cj` → slug 是 `azureOpenaiChatgpt`
- 函数名 = `getXxxPost()` 格式，驼峰

### 分类处理
- **博客文章** → 创建 `.cj`，用 safari-cdp 读原文填入实际内容
- **GitHub 仓库/Issue/PR** → 标注"非博客文章"，保留原始链接
- **搜索页面/论坛/文档** → 标注类型，保留原始链接
- **无链接** → 标注"无链接"

### 读取网页内容（safari-cdp）
- 用 **safari-cdp 技能**（osascript 驱动 Safari）读原文
- 包括：导航到 URL、提取页面文本/图片链接、DOM 查询
- 这是为了绕过反爬（EdgeOne、Cloudflare 等）
- **不能**用 urllib/requests/curl 直接抓取（会被拦截）

### 图片流程（只有博客文章类才需要）

**完整流程:**
```
safari-cdp 导航原文 → DOM 提取文本和图片 URL → 写 .cj（含图床链接）→ 按引用列表 curl 下载图片 → avifenc 转 AVIF → 确认文件名和数量完全对齐
```

**存放位置:**
- 图片临时存放: `~/Downloads/cangjie-images/<cj文件名>/`
  - 如 `azureOpenaiChatgpt.cj` → `azureOpenaiChatgpt/`
- **不在 git 仓库里**

**图床链接格式:**
- 文章里的图床链接: `bemly-moe.5ddd.com/cangjie/<cj文件名>/xxx.avif`

**图片与文章引用必须对齐:**
- `.cj` 文件里引用的每张图（`bemly-moe.5ddd.com/cangjie/<slug>/xxx.avif`），必须在 `~/Downloads/cangjie-images/<slug>/` 里有对应的**同名文件**
- **不能多也不能少**，文件名严格一致
- 先写 `.cj` 文章确定引用了哪些图，再按引用列表去下载/转换
- 如果某张图下载失败，要如实报告，不能假装有

**转换:**
- 下载图片后用 `avifenc` 转 AVIF（`brew install libavif`）

**提交:**
- 只提交 `.cj` 代码到 git
- 图片用户自行上传图床

### 代码格式
- 遵循项目里现有 `.cj` 文件的 DSL 格式
- `raw()` 里 href 转义格式参考 `azureOpenai.cj`（不能多加反斜杠）

### 违反计划规则

**如果执行过程中需要违反或偏离计划的任何步骤，必须立即停止并向用户报告，等待用户确认后才能继续。不得擅自跳过、简化或替换计划中的步骤。**

## 风险和注意事项

1. **safari-cdp 反爬**: 部分网站可能连 Safari 也拦截，需如实记录失败
2. **图片下载失败**: 某些图床可能已失效，记录在案不伪造
3. **仓颉 DSL 转义**: raw() 中的 href 转义格式必须和现有文件一致
4. **slug 冲突**: 博客类可能有重名标题，需去重处理
5. **main.cj 注册**: 每次新增 .cj 都要同步更新 main.cj
6. **NAS 编译验证**: 最终需要在 NAS 上 cjpm build 验证语法正确
